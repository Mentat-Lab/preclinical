# AGENTS.md

## Project Rules

- No OOP. Do not introduce object-oriented programming patterns, classes, inheritance, or framework-style abstractions unless explicitly requested.
- Always be as minimal and lean as possible.
- Only write the code absolutely needed to get the task done.
- Write code in the simplest and easiest possible way to understand.
- No fancy patterns, clever abstractions, or extra architecture unless explicitly asked for.
- Think local: prefer direct repo-local changes over broad platform-style designs.
- Optimize for consistency, accuracy, and latency before everything else.
- Keep the database minimal and easy to understand as soon as possible.
- Keep everything easy to debug and test.
- Make the codebase easy for other contributors to understand, run, test, and modify.

## Current Priority

This repo is a benchmark/data-collection harness. Data collection reliability matters more than broad product polish or general platform features.
